# FORK.md — maintaining `exo-10gbe-wsl` against upstream `exo-explore/exo`

This repo is a **fork** of [exo-explore/exo](https://github.com/exo-explore/exo),
customised to run a heterogeneous CUDA(WSL2)+Mac cluster over 10GbE. It is a clean
commit series on top of an upstream merge base, so we keep it current by
**rebasing onto new upstream** rather than merging.

This file is the single source of truth for *every* way the fork diverges from
upstream and how to resolve each conflict when syncing. Keep it updated when you
change an upstream file.

## How to sync

```bash
./scripts/sync-upstream.sh        # fetch upstream, list new commits + likely-conflict files, offer rebase
# or manually:
git fetch upstream
git rebase upstream/main          # replays the fork commits; resolve conflicts (see playbook below)
uv lock                           # if uv.lock conflicted, take theirs then regenerate (never hand-merge)
uv run basedpyright && uv run ruff check && nix fmt && uv run pytest
git push --force-with-lease
```

`upstream` = `https://github.com/exo-explore/exo.git`. Merge base at time of
writing: `09f9ea31` ("libp2p -> zenoh", 2026-06-03).

## Finding fork changes

Every fork edit to an upstream file is tagged on the line above it:

```bash
grep -rn "FORK(exo-10gbe-wsl)" src/ rust/ pyproject.toml
```

When a rebase conflict shows your side (`<<<<<<< HEAD`), the `FORK(...)` marker
right there tells you what it is and why. Match it to the inventory below.

## Env flags that gate fork behaviour
| flag | effect |
|---|---|
| `ENABLE_DISAGGREGATION` | turns on the disaggregated prefill→decode path (incl. all of category C) AND auto role assignment (the master auto-links a CUDA instance as prefill for a same-model Mac decode instance). |
| `EXO_CONNECT_PEERS` / `--connect-peer` | manual peer dialing (category B). |
| `OVERRIDE_MEMORY_MB` | overrides reported node memory; wins over CUDA-VRAM/system-RAM (category D). |
| `CUDA_VISIBLE_DEVICES` | selects which GPU `_cuda_vram_bytes()` reports + pins MLX-CUDA to one GPU. |
| `EXO_NO_BATCH` | use `SequentialGenerator` (faster for single-user decode; set on the decode node). *Upstream flag, set in `run-5070ti-solo.sh`.* |
| `EXO_FAST_SYNCH` | upstream Metal fast-synch toggle. |

would then disappear from this list): **F** (cold-join catch-up), **G** (ws==1
pipeline skip), `profiling.from_cuda_vram` (**D**), `find_ip_prioritised` fallback
(**E**), and `--connect-peer` (**B**, router.py/main.py). Categories **A** (deps)
and the `ENABLE_DISAGGREGATION`-gated parts of **C** stay fork-local.
